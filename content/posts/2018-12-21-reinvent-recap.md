December started with another edition of the re:Invent conference in Las Vegas.  My colleagues and I have started a tour around dozens of cities around the world to offer a re:Invent re:Cap, check your <a href="https://www.meetup.com/find/?allMeetups=false&keywords=AWS" target="_blank">local AWS User Group on Meetup</a> for details.

In France, we covered the following cities, with the help of our partners <a href="https://www.d2si.io/" target="_blank">D2Si</a> and <a href="http://www.corexpert.net/" target="_blank">CoreExpert (by TeamWorks)</a> :

- <a href="https://twitter.com/d2si/status/1072194945803534337" target="_blank">Paris</a> (Monday, 10th December)
- <a href="https://twitter.com/d2siToulouse/status/1074746986002309121" target="_blank">Toulouse</a> (Monday, 17th December)
- <a href="https://twitter.com/d2si/status/1075096028330844163" target="_blank">Lyon</a> (Tuesday, 18th of December)

[Let me know][me] if you want to add a city and date in January.

Developer's often asked me what was my favourite announcement from the 100+ made that week.  Being an adopter and advocate of serverless computing, my favourite announcement was <a href="https://docs.aws.amazon.com/lambda/latest/dg/configuration-layers.html" target="_blank">AWS Lambda Layers</a> and <a href="https://docs.aws.amazon.com/lambda/latest/dg/runtimes-custom.html" target="_blank">Custom AWS Lambda Runtime</a>.  You can read <a href="https://aws.amazon.com/blogs/aws/new-for-aws-lambda-use-any-programming-language-and-share-common-components/" target="_blank">the blog post</a> from <a href="https://twitter.com/danilop" target="_blank">Danilo</a> for more details.

This allowed developers to bring new languages to Lambda, check out these solutions :

- a <a href="https://aws.amazon.com/blogs/compute/introducing-the-c-lambda-runtime/" target="_blank">C++</a> runtime (build by AWS)
- a <a href="https://aws.amazon.com/blogs/opensource/rust-runtime-for-aws-lambda/" target="_blank">Rust</a> runtime (build by AWS)
- a <a href="https://www.stackery.io/blog/aws-lambda-php/" target="_blank">PHP</a> runtime (provided by Stackery)
- a <a href="https://github.com/tonisuter/aws-lambda-swift" target="_blank">Swift</a> runtime (opensource, built by <a href="https://tonisuter.com/" target="_blank">Toni Suter</a> from Switzerland)
- and even a <a href="https://www.bluage.com/news/serverless-cobol" target="_blank">Cobol</a> runtime (provided by BlueAge)

And probably more to come, [let me know][me] if you have developed a custom runtime to add to this list.

See you at a re:Invent re:Cap event near you!
